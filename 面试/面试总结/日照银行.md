### 1. `==` 和 `equals` 的区别

**现场情况**:回答未录到,面试官没有深挖,推测基本过关。

**标准回答**:
- `==`:基本类型比较值;引用类型比较内存地址
- `equals`:Object 默认实现就是 `==`;String、Integer 等类重写后比较内容
- **必带的加分点**:重写 `equals` 必须同时重写 `hashCode`,否则对象放进 HashMap/HashSet 会出现"equals 相等但找不到"的问题

**改进**:可以再主动提 Integer 缓存池(-128~127 内 `==` 为 true,超出为 false),面试官会觉得你踩过坑。

---

### 2. 抽象类能用 final 修饰吗?

**现场情况**:回答未录到。

**标准回答**:不能。`abstract` 的意义是必须被继承才有用,`final` 禁止继承,语义矛盾,编译直接报错。同理,抽象方法不能用 `final`、`private`、`static` 修饰。

**改进**:顺带一句"抽象类可以有构造方法和普通方法,也可以一个抽象方法都没有",展示体系化理解。

### 3. 金额计算精度(银行必问)⭐

> 原话:"我们这边是银行项目……金额计算怎么保证准确、精度高、不丢失精度,用什么类型?"

**现场情况**:面试官自己补了"BigDecimal 的加减乘除方法知道吗",推测你答出了 BigDecimal 但细节不够。

**标准回答**:
1. **为什么不能用 double**:二进制浮点无法精确表示 0.1 这类十进制小数,`0.1 + 0.2 != 0.3`
2. **用 BigDecimal,且必须字符串构造**:`new BigDecimal("0.1")`;用 double 构造精度问题会原样带进来
3. **四则运算**:`add` / `subtract` / `multiply` / `divide`;**除法必须指定精度和舍入模式**,如 `divide(b, 2, RoundingMode.HALF_UP)`,否则除不尽直接抛 ArithmeticException
4. **比较用 `compareTo` 不用 `equals`**:`equals` 连小数位数一起比,`1.0` 和 `1.00` 不相等——经典坑
5. 数据库对应 `DECIMAL(19,4)` 之类定点类型,不用 float/double

**改进**:这是银行项目的招牌题,建议把上面 5 点练成 1 分钟的完整表述,第 3、4 点是拉开差距的地方。

### 4. 线程池:创建方式和区别 ⚠️

**现场情况**:面试官连续追问"你可以简单列举一种两种""区别可以稍微说一下嘛,它有好几个方法可以创建的"——**追问式引导通常说明回答不完整或不成体系**。

**标准回答**:
- **方式一(推荐)**:`new ThreadPoolExecutor(...)` 手动创建,7 个参数:核心线程数、最大线程数、空闲存活时间、时间单位、工作队列、线程工厂、拒绝策略
- **方式二**:`Executors` 工厂方法:`newFixedThreadPool`(固定大小)、`newCachedThreadPool`(按需扩容)、`newSingleThreadExecutor`(单线程)、`newScheduledThreadPool`(定时)
- **区别/为什么不用 Executors**(阿里规约):Fixed 和 Single 用无界队列 LinkedBlockingQueue,任务堆积会 OOM;Cached 最大线程数是 Integer.MAX_VALUE,线程爆炸
- **执行流程口诀**:来任务 → 核心线程没满就建线程 → 满了进队列 → 队列满了扩到最大线程数 → 还不行走拒绝策略(四种:Abort 抛异常 / CallerRuns 调用者自己跑 / Discard 静默丢弃 / DiscardOldest 丢最老的)

**改进**:按"两种方式 → 为什么推荐手动 → 7 参数 → 执行流程"的顺序背成套路,别等面试官一点点挤牙膏。

### 5. List、Set、Map 的区别

**标准回答**:List 有序、可重复;Set 不可重复(HashSet 无序、LinkedHashSet 保插入序、TreeSet 排序);Map 存 key-value,key 不可重复。

---

### 6. List 的常用实现类 ⚠️

**现场情况**:面试官又是"你可以说个一两种嘛"的引导语气,推测答得慢。

**标准回答**:
- `ArrayList`:底层数组,查询 O(1),中间增删要移动元素;扩容 1.5 倍
- `LinkedList`:底层双向链表,头尾增删快,按下标查询要遍历
- `Vector`:线程安全但方法全加锁,性能差,基本弃用

**改进**:这属于送分题,答不流利很伤印象——集合这块整体过一遍。

---

### 7. 数组转 List 用什么方法

**标准回答**:`Arrays.asList(arr)`。**必须主动说坑**:返回的是 Arrays 内部的定长视图,`add/remove` 会抛 UnsupportedOperationException;要可变列表用 `new ArrayList<>(Arrays.asList(arr))`,Java 8 也可以 `Stream.of(arr).collect(Collectors.toList())`。

**改进**:只答 `Arrays.asList` 是 60 分,说出定长坑才是这道题真正想考的。

---

### 8. MyBatis:属性名和字段名不一致怎么处理

**标准回答**:三种——① SQL 里起别名 `user_name AS userName`;② `<resultMap>` 显式映射;③ 全局配置驼峰转换 `mapUnderscoreToCamelCase=true`(user_name 自动映射 userName)。

---

### 9. MyBatis 动态 SQL 标签,举两三个

**标准回答**:`<if>`(条件拼接)、`<where>`(自动处理 and/where)、`<foreach>`(批量插入、in 查询)、`<set>`(动态 update)、`<choose>/<when>/<otherwise>`(多选一)、`<trim>`。

**改进**:举例时带一句实际用法,比如"in 查询用 foreach 遍历 id 集合",比干背标签名有说服力。

### 10-11. 索引:创建命令 + 优缺点

**现场情况**:面试官追问"index 后面跟的是什么,是名称还是列名",推测语法没答清。

**标准回答**:
- 命令:`CREATE INDEX idx_user_name ON user(name);` 或 `ALTER TABLE user ADD INDEX idx_user_name(name);`——**index 后面跟的是索引名称**,括号里才是列名
- 优点:B+ 树结构,大幅加快查询和排序
- 缺点:占磁盘空间;增删改要同步维护索引,写入变慢;数量不宜多
- 什么列适合建:经常出现在 where/order by、区分度高的列;性别这种低区分度列不适合

**改进**:能带出"B+ 树""最左前缀原则""索引失效场景(函数、隐式转换、like '%x')"任意一个,就超出这个岗位的平均线了。

### 12-13. Linux:常用命令、部署、定时任务 ❌ 最大失分点

**现场情况**:面试官明确说"**这块没接触过是吧**"——当场暴露空白,这是全场唯一被钉死的短板。

**标准回答(背熟这 15 条就够应付)**:
```bash
cd / ls / pwd / mkdir / rm -rf        # 目录文件
cat / tail -f app.log / grep "ERROR"  # 看日志(tail -f 实时追踪必会)
ps -ef | grep java                    # 找进程
kill -9 <pid>                         # 杀进程
top / free -h / df -h                 # CPU内存磁盘
chmod +x xx.sh                        # 权限
nohup java -jar app.jar > log.txt 2>&1 &   # 部署启动(必会)
crontab -e / crontab -l               # 定时任务:编辑 / 查看
# cron 表达式五位:分 时 日 月 周,如 0 2 * * * 每天凌晨2点
```

**改进**:面试话术也要准备——之前项目有运维/CI 兜底,但自己会基本操作;然后立刻把 tail、nohup、crontab 说出来止损,不要让"没接触过"成为句号。

### 15. 微服务:Spring Cloud 还是单体?网关、注册中心用过吗

**现场情况**:录音里面试官接话"应该是做的负载""它本身没有用网关之类的"——推测你答了项目是微服务但没用全家桶组件,答得比较含糊。

**标准回答**(每个组件一句话):
- 注册中心(Nacos/Eureka):服务启动时注册地址,调用方按服务名找实例
- 网关(Gateway):统一入口,做路由转发、鉴权、限流
- 负载均衡(Ribbon/LoadBalancer):从多实例中按策略选一个
- 远程调用(Feign):声明式 HTTP 调用,写接口像调本地方法
- 配置中心(Nacos/Apollo):配置集中管理、动态刷新

**改进**:哪怕项目没用过,概念题也应该能一句话一个说清楚;准备一段"我们项目是 XX 架构,服务间调用走 XX,负载在 XX 层做"的真实描述,含糊最伤。
